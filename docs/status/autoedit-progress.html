Plain-English glossary

Definitions for the technical and project terms used in this dashboard. Open a group to browse its terms.

Speech & AI

VAD — Voice Activity Detection

Detects time ranges that contain speech rather than silence or background noise. AUTOEDIT’s current fallback combines VAD with audio-level evidence, but it cannot reliably identify the physical speaker when microphones bleed into each other.

Whisper

OpenAI’s automatic speech-recognition model family. It turns spoken audio into text with timestamps; it does not, by itself, reliably identify who said each word.

WhisperX

A speech-processing pipeline built around faster Whisper transcription, word-level alignment, and optional speaker diarization. AUTOEDIT plans to use its timed speaker turns as stronger evidence for camera choices.

WhisperX service / worker

A separate, private GPU-backed service planned for AUTOEDIT. It reads synchronized analysis audio, runs transcription, word alignment, and diarization, then returns versioned JSON results to the main app. It is kept separate so GPU AI work cannot destabilize the public web application.

ASR — Automatic Speech Recognition

The general technology that converts speech into written text. WhisperX uses Whisper/faster-whisper as its ASR component.

Word alignment

Refines transcript timing so individual words have accurate start and end positions on AUTOEDIT’s master timeline.

Speaker diarization

Separates an audio recording into anonymous speaker turns—essentially “who spoke when.” It may output labels such as SPEAKER_00, which still need mapping to real people and cameras.

pyannote

A machine-learning toolkit commonly used by WhisperX for speaker diarization. Some of its models require approved Hugging Face access and a private token.

Speaker mapping

The per-project link between an anonymous diarization label, a real participant, and that participant’s camera. AUTOEDIT can suggest this mapping but must expose uncertainty rather than guess.

LLM — Large Language Model

A text model used for semantic tasks such as topic grouping, editorial suggestions, or title generation. It is separate from the WhisperX speech-analysis service.

Mock backend

A deterministic placeholder used for safe development and tests. It exercises the application flow without claiming to perform real transcription, diarization, or AI reasoning.

AI artifact

A versioned, inspectable JSON result containing model details, transcript segments, timed words, speaker turns, confidence, provenance, warnings, and errors. It makes AI output auditable and replaceable.

Media & editing

Energy envelope

A simplified graph of audio loudness over time. AUTOEDIT compares these graphs to synchronize recordings and, in the fallback route, to estimate activity.

Cross-correlation audio sync

The automatic process of sliding two energy envelopes against each other to find the time offset where they match best. This is AUTOEDIT’s required sync method; it avoids manual timeline nudging.

Program audio / master clock

The authoritative mixed audio track used for review playback and timing. Silent camera proxies follow this clock so angle changes do not interrupt or duplicate sound.

Proxy

A smaller, easier-to-stream copy of source video. Editing decisions refer back to the originals; the proxy improves browser playback and remote review.

FPS — Frames Per Second

The video frame rate. Rates such as 23.976 are stored precisely as a fraction—24000/1001—to prevent timing drift.

Timeline / master timeline

The single synchronized time basis used by video, program audio, transcripts, speaker turns, notes, and edit decisions.

Cut decision / CDL

A timed instruction saying which camera should be shown and why. In this project, CDL means the internal Cut Decision List, not the colour-grading format that sometimes uses the same initials.

LUT — Look-Up Table

A colour transform used to preview a camera with a particular grade or look. It changes display appearance, not the underlying edit timing.

NLE — Non-Linear Editor

Professional editing software such as DaVinci Resolve or Final Cut Pro, where editors can refine AUTOEDIT’s exported timeline.

FCPXML

Final Cut Pro XML, a structured interchange format used to transfer timelines, clips, timing, and metadata into compatible NLEs.

EDL — Edit Decision List

A simpler, older timeline interchange format. It is broadly compatible but carries less detail than FCPXML.

OTIO — OpenTimelineIO

An open-source library and interchange model for representing editorial timelines. AUTOEDIT’s approved Stage 8.3 plan treats it as an optional adapter path that must preserve exact frames, clips, and note markers without changing the existing direct FCPXML or EDL defaults.

Shot reason

The auditable explanation attached to a camera choice—for example, identified speaker activity, a hold, overlap handling, or an anti-jitter rule.

Platform, delivery & testing

API / payload

An API is the structured interface between software components. A payload is the data sent through it—for example, frame-rate or synchronization settings submitted by the browser.

MySQL / mysqldump

MySQL is the database server used by AUTOEDIT. mysqldump is its command-line backup tool; a release must produce a complete, readable backup before it changes the live application.

NPM — Nginx Proxy Manager

The reverse-proxy service that fronts AUTOEDIT on the public HTTPS domain. It terminates TLS and forwards approved requests to the private app, so an NPM 200 is a public-proxy reachability check.

Least-privilege backup

A backup method that uses only the database permissions it needs, without granting broad administrator access. The Publisher must fail closed if the account cannot create a clean backup.

GPU / CUDA / VRAM

The GPU performs accelerated AI computation; CUDA is NVIDIA’s software platform for that computation; VRAM is the GPU’s dedicated memory, which limits model and batch size.

VAAPI

Linux hardware-video acceleration used by AUTOEDIT’s main app for Intel proxy encoding. It is separate from the NVIDIA/CUDA path planned for WhisperX.

TLS

Transport Layer Security—the encryption behind HTTPS. It protects credentials and review traffic while data travels over the network.

XSS — Cross-Site Scripting

A browser security flaw where saved user content can execute as code. Stage 7.4 must prove that text such as <script> is displayed safely rather than run.

Commit / branch / worktree

A commit is a recorded set of changes; a branch is an independent line of development; a worktree is a separate checkout that lets an agent work on a branch without altering the main checkout.

Kanban card

A durable work item assigned to an agent, containing scope, evidence requirements, status, comments, and the resulting handoff.

Kanban claimant / heartbeat

The dispatcher process that atomically takes ownership of an eligible card. A heartbeat is a recent liveness signal from that owner; a card marked running is not treated as a completed or verified result.

Acceptance gate

A required review or test checkpoint that must pass before work can advance. A reported result is not independently accepted until the appropriate reviewer verifies it.

TEST_PASS / TEST_FAIL

The independent Tester’s durable verdict for an exact commit. TEST_FAIL blocks integration even when most checks pass; the defect must be corrected, re-reviewed, and retested.

Regression test

A test that ensures an existing capability still works after a change. A combined regression runs both feature-specific checks and the broader suite after integration.

Accepted / integrated / merged / deployed / rolled back

Accepted means the required review gate passed; integrated means exact changes were combined and tested together; merged means they entered the main Git branch; deployed means a candidate was put on the live system; rolled back means the prior known-good version was restored. These states are independent and must not be collapsed.

Release state flags

mutation_started says whether a live change began; candidate_live says whether the new image may still be serving; rollback_required says recovery is mandatory; and rollback_verified is true only after the prior image and health checks are independently proved.

P1 / P2

Labels for the two current UI implementation slices: P1 is Home/Ingest and P2 is the review Player. They are workflow slices, not numbered AUTOEDIT product stages.

Product stage vs AI phase

The original product roadmap uses numbered Stages 3–9; the newer WhisperX modernization roadmap separately uses Phases 0–8. Stage 7.4 and AI Phase 4 can therefore both be current at the same time.

Natural-language intent

The parser that turns a plain-English request, such as a requested time range or topic filter, into a structured AUTOEDIT sub-edit command. Its deterministic tests are separate from the browser notes acceptance.
